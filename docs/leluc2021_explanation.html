Control Variate Selection for Monte Carlo Integration

A complete walkthrough of
Leluc, Portier & Segers (2021), Statistics and Computing 31:50 — DOI 10.1007/s11222-021-10011-z

Prepared for the ENSAE M1 project: Control variates for a GARCH model — Question 3

0   The Big Picture

We want to estimate an integral $P(f) = \int f(x)\,P(dx)$ using a Monte Carlo sample $X_1,\ldots,X_n \sim P$. The basic MC estimate is $\hat{\alpha}^{\mathrm{mc}}_n(f) = \frac{1}{n}\sum_i f(X_i)$ with variance $n^{-1}\sigma_0^2(f)$.

We have $m$ control variates $h_1,\ldots,h_m$ with $P(h_k)=0$. For any coefficient vector $\beta \in \mathbb{R}^m$, the corrected estimator $P_n(f - \beta^T h)$ is still unbiased. The oracle optimal $\beta^\star$ minimises the variance, and the resulting oracle estimator achieves variance $n^{-1}\sigma^2_m(f) \leq n^{-1}\sigma^2_0(f)$.

Central Question Since $\beta^\star$ is unknown, how do we estimate it from data without introducing too much extra error — especially when $m$ is large?

1   OLS Monte Carlo

Replace $P$ by the sample measure $P_n$ in the normal equations. This gives the OLS estimator:

$$\left(\hat{\alpha}^{\mathrm{ols}}_n(f),\, \hat{\beta}^{\mathrm{ols}}_n(f)\right) \in \arg\min_{(\alpha,\beta)\in\mathbb{R}\times\mathbb{R}^m} \left\|f^{(n)} - \alpha\mathbf{1}_n - H\beta\right\|_2^2, \tag{OLS}$$

where $H = (h_j(X_i))_{i,j}$ is the $n\times m$ matrix of evaluated control variates. The intercept $\hat{\alpha}^{\mathrm{ols}}_n(f)$ is the ZV estimate of $P(f)$.

Two problems with OLS when $m$ is large
  1. Computational cost: $\mathcal{O}(nm^2 + m^3)$ — the $m^3$ term grows catastrophically (e.g.\ $m=1000 \Rightarrow m^3 = 10^9$ operations just to invert $H_c^TH_c$).
  2. Numerical instability / overfitting: when $m$ is comparable to $n$, the Gram matrix $H_c^TH_c$ becomes ill-conditioned and OLS over-fits the noise.

The paper proves (Theorem 1) a concentration inequality for the OLS error: with probability at least $1-\delta$,

$$\left|\hat{\alpha}^{\mathrm{ols}}_n(f) - P(f)\right| \leq \sqrt{\frac{2\log(8/\delta)\,\tau^2}{n}} + 58\sqrt{\frac{Bm\log(8m/\delta)\log(4/\delta)\,\tau^2}{n^2}},$$

where $B = \sup_x h(x)^T G^{-1} h(x)$ (a measure of the control variate complexity) and $\tau^2$ is a sub-Gaussian variance parameter of the residual. The first term is the oracle rate; the second is the extra cost of estimating $\beta$. For this bound to be tight, one needs $mB\log(m) = \mathcal{O}(n)$ — i.e.\ $m$ must not be too large.

2   LASSO Monte Carlo

The LASSO estimator adds an $\ell_1$ penalty to the OLS objective:

$$\hat{\beta}^{\mathrm{lasso}}_n(f) \in \arg\min_{\beta\in\mathbb{R}^m} \frac{1}{2n}\left\|f^{(n)}_c - H_c\beta\right\|_2^2 + \lambda\|\beta\|_1, \tag{LASSO}$$ $$\hat{\alpha}^{\mathrm{lasso}}_n(f) = P_n\!\left[f - \hat{\beta}^{\mathrm{lasso}}_n(f)^T h\right].$$

The $\ell_1$ penalty forces most coefficients to zero, effectively selecting the most useful control variates. This addresses both problems with OLS: fewer non-zero coefficients means faster computation and less overfitting.

The paper proves (Theorem 2) that the LASSO error bound replaces the total number $m$ of variates by the active number $\ell^\star = |\mathrm{supp}(\beta^\star)|$:

$$\left|\hat{\alpha}^{\mathrm{lasso}}_n(f) - P(f)\right| \leq \sqrt{\frac{2\log(8/\delta)\,\tau^2}{n}} + 476\ell^\star\log(8m/\delta)\,(U_h^2/\gamma^\star)\,\frac{\tau^2}{n},$$

where $U_h = \sup_x \max_k |h_k(x)|$ and $\gamma^\star$ is the smallest eigenvalue of the Gram matrix restricted to active variates. When only $\ell^\star \ll m$ variates are truly useful, this is dramatically tighter than the OLS bound.

Choice of $\lambda$

The regularisation parameter $\lambda$ controls the number of selected variates. Two practical methods are proposed:

  • $K$-fold cross-validation (Algorithm 1): standard but computationally expensive.
  • Dichotomic search (Algorithm 2): enforces the number of selected variates to lie in $[c_1\sqrt{n},\, c_2\sqrt{n}]$ (with $c_1=3$, $c_2=12$ recommended). Much faster and motivated by the theoretical bound.

3   LSLASSO — The Best of Both Worlds

The LSLASSO (Least-Squares LASSO) method combines:

1 Selection step (LASSO on a subsample of size $N \leq n$): Run LASSO on a smaller subsample $X_1,\ldots,X_N$ to identify the active set $\hat{S}_N = \mathrm{supp}(\hat{\beta}^{\mathrm{lasso}}_N(f))$.

2 Estimation step (OLS on the full sample, active variates only): Run OLS on the full sample $X_1,\ldots,X_n$ using only the $\hat{\ell}$ selected variates.

The theoretical guarantee (Theorem 4) is essentially the same as for OLS with the true active set — the LASSO correctly recovers $S^\star$ with high probability (Theorem 3), after which the LSLASSO reduces to the optimal OLS.

Why use a subsample for step 1? The LASSO selection only needs to identify which variates are useful, not estimate their coefficients precisely. A subsample of size $N = \lfloor 15\sqrt{n}\rfloor$ is enough for selection, and reduces the first-step cost from $\mathcal{O}(nD)$ to $\mathcal{O}(ND)$ where $D$ is the number of coordinate descent iterations.

4   Applying This to Our GARCH Problem (Q3)

Degree-2 control variates

From Mira et al. (2013) Eq. (8), using $P(\theta) = \mathbf{a}^T\theta + \frac{1}{2}\theta^T B\theta$: $$\tilde{f}(\theta) = f(\theta) - \tfrac{1}{2}\mathrm{tr}(B) + (\mathbf{a} + B\theta)^T \mathbf{z}(\theta).$$ For $d=3$ (GARCH), this gives $m = \frac{1}{2}d(d+3) = 9$ control variates:

  • Linear (3): $z_1, z_2, z_3$
  • Quadratic (6): $\omega_j z_k$ for $j \leq k$

While $m=9$ is not large enough for OLS to be a problem in our 3-parameter case, the LASSO approach is still valuable because:

  • Not all 9 quadratic terms may be equally useful — LASSO identifies the most relevant ones
  • The demonstration generalises to higher-dimensional models
  • LASSO avoids overfitting when the sample has high autocorrelation (low ESS)

5   Key Takeaways

Summary
  • OLS is optimal when $m \ll n$ but collapses when $m \sim n$.
  • LASSO selects the few truly useful variates, keeping computation cheap and avoiding overfitting.
  • LSLASSO achieves the oracle OLS rate once LASSO correctly selects the active set.
  • The dichotomic search (target $\sqrt{n}$ selected variates) is the practical recommendation.
  • In the Bayesian inference examples (Section 5), LASSO/LSLASSOX consistently outperform OLS when $m$ is large — by orders of magnitude in some cases.
